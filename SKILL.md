# Sandeep Frontend

Use this as Sandeep's front-end design skill. It should help Codex build interfaces that feel intentionally designed, light, readable, and specific to the subject instead of defaulting to generic AI/SaaS patterns.

## Inspirations

This skill is inspired by Anthropic's `frontend-design` skill for its subject-grounded process and by Every.to for its light editorial/product taste. Declare that inspiration once when useful, then treat the rest as Sandeep's own design language.

## Workflow

1. Read `references/design-language.md` before visual decisions.
2. Name the product, audience, primary job, and visual mode.
3. Choose a visual mode that fits the brief.
4. Create a compact token plan: colors, type roles, spacing, layout, components, and copy voice.
5. Build light-first UI. Do not add dark mode unless the user explicitly asks.
6. Verify mobile and desktop screenshots for type fit, hierarchy, and non-generic feel.

## Default Taste

- Prefer light, precise, content-led interfaces.
- Prefer compact hierarchy over giant headings.
- Use pills/chips where they act as controls, filters, metadata, status, or CTAs.
- Use cards only for repeated items or genuinely framed tools.
- Spend distinctiveness in one memorable place, then keep the rest disciplined.
- Let the subject matter decide the final palette and layout language.

## Pairing

If another frontend skill is active, use its implementation workflow where helpful, then apply this skill for Sandeep's taste, constraints, and visual critique.
