# Sandeep Frontend

Use this as Sandeep's front-end and visual artifact design skill. It should help agents build interfaces, HTML reports, dashboards, visualizations, slides, and other designed outputs that feel intentional, light, readable, and specific to the subject instead of defaulting to generic AI/SaaS patterns.

## Inspirations

This skill is inspired by Anthropic's `frontend-design` skill for its subject-grounded process and by Every.to for light editorial/product taste. Declare that inspiration once when useful, then treat the rest as Sandeep's own design language.

## Workflow

1. Read `references/design-language.md` before visual decisions.
2. Name the artifact type, audience, primary job, and visual mode.
3. Choose a mode that fits the brief: app, website, dashboard, HTML report, data visualization, slide deck, document-like artifact, or creative tool.
4. Create a compact token plan: colors, type roles, spacing, layout, components, chart/visual language when relevant, and copy voice.
5. Build light-first unless the user explicitly asks otherwise.
6. Verify the actual output format: responsive screenshots for web, rendered pages for reports, slide thumbnails for decks, and chart legibility for visualizations.

## Default Taste

- Prefer light, precise, content-led interfaces.
- Prefer compact hierarchy over giant headings.
- Use pills/chips where they act as controls, filters, metadata, status, or CTAs.
- Use cards only for repeated items or genuinely framed tools.
- Use charts, tables, diagrams, screenshots, or slide layouts when they carry the work better than prose.
- Spend distinctiveness in one memorable place, then keep the rest disciplined.
- Let the subject matter decide the final palette and layout language.

## Pairing

If another frontend skill is active, use its implementation workflow where helpful, then apply this skill for Sandeep's taste, constraints, and visual critique.
